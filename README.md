# jlcpcb-mapper

LLM-assisted JLCPCB part mapping for KiCad schematics. Reads empty-Footprint
symbols, queries the `kicad-jlcpcb-tools` SQLite catalog, uses Claude Code to
pick a good LCSC per component group, then assigns a KiCad built-in footprint
(falling back to an on-the-fly EasyEDA download when needed).

## Install

```bash
pipx install jlcpcb-mapper
# or for development:
uv venv && source .venv/bin/activate
uv pip install -e ".[dev]"
```

## Prerequisites

- KiCad 9 project
- [`kicad-jlcpcb-tools`](https://github.com/bouni/kicad-jlcpcb-tools) installed (for `parts.db`)
- `claude` CLI on PATH (Claude Code)

## Usage

```bash
cd /path/to/my-kicad-project
jlcpcb-mapper init                         # scaffold jlcpcb-mapper.yaml
jlcpcb-mapper map my-project.kicad_pro     # map empty-footprint components
jlcpcb-mapper verify my-project.kicad_pro  # recheck stock/EOL changes
```

## Config

See `jlcpcb-mapper.yaml` generated by `init`. Key sections:

- `selection.defaults.<category>.package` — hard package default (e.g. `0402` for resistors)
- `selection.min_stock` — SQL filter minimum stock
- `hints` — natural-language guidance for the LLM (e.g. "avoid Y5V dielectric")
- `kicad_footprint_map_overrides` — project-specific (category, package) → footprint mappings

## Safety

- Refuses to run on a dirty git tree (override: `--force`)
- Atomic schematic writes with timestamped backups in `.jlcpcb-mapper/backups/`
- Detailed JSON run log at `.jlcpcb-mapper/run-<ts>.json`

## Flags

**`map`** — `--config PATH`, `--non-interactive`, `--force`, `--allow-stale-db`,
`--fill-lcsc-only`, `--include-dnp`, `--apply-2nd-pass-suggestions`

**`verify`** — `--config PATH`, `--non-interactive`, `--force`, `--allow-stale-db`
